类的启动过程--OC

By Vodka

发表于 2014-07-07

  1. 只要程序启动就会将所有的类的代码加载到内存中，放到代码区
  2. load方法会在当前类被加载到内存的时候代用，有且仅会调用一次

  3. 如果存在继承关系，会先调用父类的load方法，然后在调用子类的load方法

  4. initialize方法当当前类第一次被使用的时候会调用(创建类对象的时候)

  5. initialize方法在整个程序的运行过程中只会被调用一次，无论你使用多少次这个类都只会调用一次
  6. initialize用于对某一类进行一次性的初始化

  7. 存在继承关系，initialize和load一样

  8. dealloc方法在对象引用计数器为0的时候会自动调用，即销毁对象

SVN常用命令集合

By Vodka

发表于 2014-07-04

  1. SVN常用命令集

    svn info 查询svn根目录的信息，这个命令当你想知道某个svn目录是从哪个服务器上下来的时候特别有效
    svn add 增加一个文件到根目录里，注意要用svn commit上传这个修改
    svn status 查询当前目录下文件修改的情况，a表示增加，M表示修改
    svn diff 查看本目录下所有的文件有哪些区别，当然可以指定到文件名
    svn commit -m “fix bug” file上传某个文件的修改，并增加注释
    svn ci 上传所有的修改，会提示你添加修改记录
    svn log file 查询某个文件的修改记录
    svn checkout 从svn服务器上去一个目录，带svn信息
    svn export 从svn服务器上取出一个目录，仅源文件，没有讨厌的.svn信息
    svn revert 回滚本地所有的未上传的所有修改，慎用，回复该本地所有的修改操作。可一次回滚一个目录或者文件
    svn revert file —depth=infinity 回滚该目录下的所有文件
    svn diff -r3 rules.txt 将本地的working目录下的文件和服务器的r3版本之间进行比较
    svn diff -r 3:2 rules.txt 比较服务器上得r2版本和r3版本

一、沙盒目录结构

每个APP的沙盒都有相似的目录结构，如图：

获取应用程序目录的路径，如下：

该目录中包含：

  1. Documents：应该将所有的应用程序数据文件写入到这个目录下。这个目录用于存储用户的数据，该路径可以通过配置实现iTunes共享文件，可被iTunes备份。
  2. AppName.app：这是应用程序的程序包目录，包含应用程序的本身。由于应用程序必须经过签名，所以在运行时不能对这个目录中的内容进行修改，否则可能会使程序无法启动。
  3. Library：可创建子文件夹，用来放置你希望被备份但是不希望被用户看到的数据，该路径下的文件夹，除Caches以外，都会被iTunes备份。包含：
    1. Preferences：包含应用程序的偏好设置文件。不应该直接创建偏好设置文件，而是应该使用NSUserDefaults类来取得和设置应用程序的偏好。
    2. Caches：用于存放应用程序专用的支持文件，保存应用程序再次启动过程中需要的信息。
  4. tmp：这个目录用于存放临时文件，保存应用程序再次启动过程中不需要的信息。该目录不会被iTunes备份。

三、NSSearchPathForDirectoriesInDomains

该方法用于查找目录，返回指定范围的制定名称的目录的路径集合。有三个参数：

  1. directory：NSSearchPathDirectory类型的枚举值，表明我们要搜索的目录名称
  2. domainMask：NSSearchPathDomainMask类型的枚举值，指定搜索的范围，这里的NSUserDomainMask表示搜索的范围限制于当前应用的沙盒目录。还可以写成NSLocalDomainMask(表示/Library)、
    NSNetworkDomainMask(表示/Network)等。
  3. expandTilde：布尔值，表示是否展开波浪线~。在iOS中~的全写形式是/User/userName，该值为YES表示写成全写形式，为NO表示直接写成~.
